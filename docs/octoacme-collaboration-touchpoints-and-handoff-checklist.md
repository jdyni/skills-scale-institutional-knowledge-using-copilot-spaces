# OctoAcme — Collaboration Touchpoints & Handoff Checklist

## Purpose
Make cross-role collaboration and handoffs explicit so ownership gaps are caught early.

## RACI-lite (fill at kickoff/planning)
| Activity | PM | PdM | Developers | QA/Testing | Stakeholders | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| Problem framing and success metrics | A | R | C | C | C | |
| Backlog prioritization and scope | C | A/R | C | C | C | |
| Build and implementation | C | C | A/R | C | I | |
| Test strategy and acceptance validation | C | C | C | A/R | I | |
| Release readiness and communication | A/R | C | C | C | C | |

_Legend: R = Responsible, A = Accountable, C = Consulted, I = Informed_

## Collaboration Touchpoints
- [ ] PM + PdM agree milestones, owners, and decision cadence
- [ ] User research/design findings reviewed with PdM, Developers, QA
- [ ] Developers + QA confirm test approach for acceptance criteria
- [ ] Technical Writer confirms documentation scope and owners
- [ ] DevOps + Developers confirm deployment/rollback approach
- [ ] Stakeholder update cadence and communication owner confirmed

## Handoff Checklist (Execution to Release)
- [ ] Acceptance criteria are met and traceable to backlog items
- [ ] QA results and unresolved risks are documented
- [ ] Documentation updates and release notes are complete
- [ ] Monitoring, alerting, and rollback plan are validated
- [ ] Final go/no-go decision and approvers are documented
